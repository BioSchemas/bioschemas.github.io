Biotea 2 Bioschemas - Introduction

Biotea defines a model for metadata, references, content and names entities recognized in the text. Bioschemas profiles for scholarly publications focus on metadata only.

    Biotea 2 Bioschemas - Use cases

    Findability

    Easily finding scholarly publication available on the web for the research community, related to entities used in their domain (via annotations with specialized vocabularies)

    Accessibility

    As publications are marked up with schema.org/Bioschemas, their key elements can be easily accessible by people and machines. Currently, the Journal Article Tag Suite is the default format supported by main publishers (at least in the Life Sciences domain. The JATS XML format is a comprehensive one including metadata, abstract, content and references. The format though lacks support for Linked Data. JSON is also supported via web services by some repositories such as PubMed -- see e-utils tools, and Europe PMC, see RESTful services. Biotea adds a semantic layer on top of JATS as well as an easy way to access content, via annotations, with no need to expose it all.

    Summarization

    A summarization similar to those for movies would become possible thanks to schema.org/Bioschemas markup, providing a quick view regarding a publication.

    Interoperability

    Interoperability refers to the use of common vocabs and annotation schemas for data exchange purposes. For instance, by knowing that a protein is a protein then automatic links can be built to specific dbs. By using common annotation schemas then annotations can easily be exchanged (use cases for such tasks are part of the annotation ontology or annotation framework). The exchange should also make it easier to identify the location of the annotation in the text.
